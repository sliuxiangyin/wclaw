# 百度千帆 Coding Plan 新增支持 GLM-5.1 和 DeepSeek-V4-Flash

近日，百度千帆平台对其 Coding Plan 进行了更新，新增了对 **GLM-5.1** 和 **DeepSeek-V4-Flash** 两款模型的支持。这一变化为开发者提供了更多样化的模型选择，以适应不同复杂度的任务需求。

## 新增模型概览

根据社区用户 kanjuu 的分享，本次更新主要新增了以下两款模型：

- **GLM-5.1**：该模型明确标注了“抵扣系数较高，仅推荐在复杂任务场景使用”。这意味着它在处理高难度、高精度的任务时表现更佳，但相应的资源消耗也更大。
- **DeepSeek-V4-Flash**：作为 DeepSeek 系列的最新版本，该模型在性能和效率上进行了优化，适合日常开发场景。

## 使用注意事项

- **模型选择**：建议开发者根据任务的实际复杂度选择合适的模型。对于简单任务，优先使用 DeepSeek-V4-Flash 以节省资源；对于复杂任务，则可选用 GLM-5.1 以获得更优结果。
- **资源消耗**：GLM-5.1 的抵扣系数较高，使用前请确认账户余额或配额充足。

百度千帆 Coding Plan 的这次更新，为开发者提供了更丰富的模型选择，尤其在处理复杂任务时有了更专业的工具。建议开发者根据实际需求合理选用，以平衡性能与成本。
