# Linux 内核现严重本地提权漏洞，影响大量系统

近日，安全研究人员披露了一个影响广泛的 Linux 内核本地提权漏洞，该漏洞允许普通用户获取系统最高权限（root），对服务器和个人设备构成严重威胁。

![](https://cdn3.ldstatic.com/optimized/4X/0/b/d/0bd035128710250b4f8e6e21622d4880927f0ae1_2_690x319.jpeg)

## 漏洞概述

该漏洞存在于 Linux 内核的某个核心子系统中，攻击者通过精心构造的本地程序，可以利用内核内存管理缺陷，实现权限提升。一旦成功利用，攻击者可以从普通用户账户直接获得 root 权限，完全控制系统。

## 影响范围

- **内核版本**：影响 Linux 内核 5.x 至 6.x 多个版本
- **受影响系统**：包括主流发行版如 Ubuntu、Debian、CentOS、RHEL、Fedora 等
- **潜在风险**：云服务器、桌面系统、嵌入式设备均可能受影响

![](https://cdn3.ldstatic.com/optimized/4X/1/1/c/11c8e1d1f8c2825841f06dd422b6377a66b244a6_2_690x331.png)

## 技术细节

漏洞利用流程大致如下：

1. **触发条件**：攻击者通过特定系统调用序列触发内核中的竞态条件
2. **内存破坏**：利用内核堆内存的释放后使用（Use-After-Free）漏洞
3. **权限提升**：篡改内核关键数据结构，将当前进程的权限提升为 root

![](https://cdn3.ldstatic.com/optimized/4X/6/0/8/608aed30c71d47a897d18d8ab73986635e93f142_2_689x286.png)

## 缓解措施

### 立即行动

- **更新内核**：各发行版已发布安全更新，请尽快执行系统更新
  - Ubuntu/Debian：`sudo apt update && sudo apt upgrade`
  - CentOS/RHEL：`sudo yum update kernel`
- **重启系统**：内核更新后需重启生效

### 临时防护

- **限制本地用户**：减少不必要的本地用户账户
- **启用 SELinux/AppArmor**：增强访问控制
- **监控异常行为**：关注系统日志中的可疑提权尝试

![](https://cdn3.ldstatic.com/optimized/4X/a/f/2/af29762e2699197a6c7d000b49db5bba6bfa06de_2_674x500.jpeg)

## 总结

该本地提权漏洞危害严重，但利用需要本地访问权限。对于多用户服务器和云环境，风险尤为突出。建议所有 Linux 用户和管理员立即评估受影响情况，并优先安排内核更新。

**安全无小事，及时修补是关键。**
