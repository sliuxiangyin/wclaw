# Dirty Frag: 又一个Linux 提权漏洞

近日，安全研究员 Hyunwoo Kim（@v4bel）发现并公开了一个名为 **Dirty Frag** 的 Linux 内核提权漏洞。该漏洞属于逻辑漏洞类，通过串联特定的内核组件缺陷，攻击者可以在主流 Linux 发行版上稳定获取 root 权限。

## 漏洞概述

Dirty Frag 是 **Dirty Pipe** 和 **Copyfail** 等漏洞所属“虫类”的扩展案例。与许多依赖竞态条件的漏洞不同，Dirty Frag 是一个**确定性逻辑漏洞**，不依赖时间窗口，因此：

- 无需复杂的竞态条件
- 内核在漏洞利用失败时不会崩溃
- 成功率非常高

该漏洞通过以下两个内核组件进行串联利用：

- **xfrm-ESP Page-Cache Write**
- **RxRPC Page-Cache Write**

目前，该漏洞的详细技术信息和时间线已在以下链接公开：

由于禁运已被打破，目前**没有官方补丁或 CVE 编号**。经过与 `linux-distros@vs.openwall.org` 维护者协商并应其要求，Dirty Frag 文档已正式发布。

## 漏洞利用演示

以下命令可用于在受影响系统上快速验证漏洞（请仅在授权环境中测试）：

```bash
git clone https://github.com/V4bel/dirtyfrag.git && cd dirtyfrag && gcc -O0 -Wall -o exp exp.c -lutil && ./exp
```

## 影响与建议

Dirty Frag 影响主流 Linux 发行版，建议系统管理员和安全团队：

1. **密切关注**上游内核安全公告和补丁发布
2. **限制本地用户权限**，降低被利用风险
3. **启用内核安全模块**（如 SELinux、AppArmor）增加攻击难度
4. **及时更新**内核版本，待补丁发布后尽快部署
