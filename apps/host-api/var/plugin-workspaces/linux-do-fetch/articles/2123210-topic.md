# DeepSeek V4 Pro带来的惊喜：一次AI排错实战的意外收获

> 当GPT和GLM都在“钻牛角尖”时，DeepSeek却用优雅的方式解决了问题

## 一次令人沮丧的排错经历

最近在排查模型训练代码时，遇到了一个棘手的fp8训练导致NaN的问题。一直以来，我都很信任GPT的严谨和认真，这次也不例外——耐心地和GPT交流想法思路，希望能快速定位问题。

然而，时间一分一秒过去，我却感到越来越不对劲。

### GPT的“死脑筋”

我和GPT一致认为需要定位出问题的网络节点，甚至已经锁定了具体网络层。但GPT像着了魔一样，要么反复验证这个结论，要么建议我直接关掉整个fp8，就是没有针对问题思考修复方案。

等我反应过来时，一整天时间已经过去了。

![](https://cdn3.ldstatic.com/original/4X/6/4/d/64d967d7d0f167975033ee49b2742cfbe735d152.png)

### GLM的“耗时间”

尝试换上GLM 5.1，它也能定位到问题位置。但紧接着，它做了一个非常耗时的实验，一不注意就把我五小时的上限吃完了。

![](https://cdn3.ldstatic.com/original/4X/c/f/7/cf7ffa462a0a8f3f0da9946752dccb4e715d58ed.png)

## 转机：DeepSeek V4 Pro Max登场

在几乎要放弃的时候，我决定试试DeepSeek V4 Pro Max。结果令人惊喜：

- **定位错误**：快速找到问题根源
- **思考方案**：提出针对性修复思路
- **插入代码**：一气呵成，直接给出解决方案

虽然最后工具触发失败，但问题真的解决了——一个很优雅的scale相乘，再也没有NaN问题了。

![](https://cdn3.ldstatic.com/original/4X/f/2/2/f22a612411153199b6b75679ec7a4ebe4de49fbe.png)

## 反思与收获

这次经历让我意识到：GPT不仅严谨认真，还有点死脑筋。我是真的没想到GPT会在钻牛角尖——看它说得头头是道，我还一直反思是不是自己的方向错了。

之前用DeepSeek不多，这次确实感觉到了它的实力。在需要快速解决问题、避免陷入无意义循环的场景下，DeepSeek展现出了独特的优势。

**总结**：AI工具各有千秋，关键是要根据任务特点选择最合适的伙伴。当遇到需要跳出固定思维框架的问题时，不妨给DeepSeek一个机会。
