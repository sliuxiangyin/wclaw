# 插件架构文档

本文从 **`@wclaw/plugin-sdk`** 视角说明 **宿主—插件架构**：职责切分、依赖注入与扩展点的正交关系。本包仅提供 **类型与少量运行时辅助类**，不包含宿主实现。

---

## 1. 分层与依赖方向

```text
┌─────────────────────────────────────────────────────────┐
│ 宿主 host-api                                           │
│  扫描 plugin.json · 校验 · import(entry) · new class   │
│  会话路由 · AI 编排 · 落库 · 调度 · MCP/LLM 网关         │
└───────────────────────────┬─────────────────────────────┘
                            │ constructor(deps)
                            │ 调用可选方法 executeTurn …
┌───────────────────────────▼─────────────────────────────┐
│ 插件 entry：export default class                         │
│  implements（概念上）PluginRuntimeExtension               │
│  · 禁止 import 宿主内部                                  │
│  · 只能通过 deps 收窄 API 访问宿主能力                    │
└─────────────────────────────────────────────────────────┘

        @wclaw/plugin-sdk（本包）
        · 契约类型 PluginRuntimeExtension / Deps / Context …
        · 工具函数 toTurnResult · toSessionRow · guard*
        · 可选基类 BasePluginRuntime（封装桥调用与 workspace）
```

- **单向依赖**：插件依赖 SDK 类型即可；SDK **禁止**依赖宿主、Fastify、SQLite。
- **契约稳定性**：宿主与插件应共同以本包 **`src/runtime-contract.ts`** 导出类型为单一事实来源，避免手写重复类型分叉。

## 2. 构造期注入：`PluginRuntimeExtensionDeps`

宿主在 **`new PluginRuntime(defaultClass, deps)`** 语义下传入 **`PluginRuntimeExtensionDeps`**：

| 成员 | 作用 |
|------|------|
| **`pluginId`** | 当前插件实例身份 |
| **`publish`** | 指向宿主事件枢纽的 **`publish`** 窄接口（`PluginHostPublishInput`），**唯一**允许的构造期 ctx 扩展之一 |
| **`workspaceDir`** | 可选；插件独占工作目录根路径，便于安全落盘（见 **`BasePluginRuntime`** 的路径校验） |
| **`ingestExternalUserTurn`** | 可选；外部进线与 UI Chat **同源**编排入口 |
| **`invokeHostMcpTool` / `releaseHostMcpContext`** | 可选；经宿主 MCP 网关调用工具与释放会话上下文 |
| **`invokeHostLlm`** | 可选；经宿主配置的 LLM 一次补全（需清单声明 **`capabilities.llm`** 且宿主启用） |

**返回值与 Hub**：插件实例方法的 **同步/异步返回值与异常** 不经由 Hub；**`publish`** 用于宿主侧多 topic 分发，与控制台 **`emitPluginActivity` + SSE** 等通路职责不同（宿主实现以项目设计文档为准）。

## 3. 扩展点正交性

以下为概念上的 **正交通路**（互不替代）：

| 扩展点 | 关注点 |
|--------|--------|
| **`executeTurn`** | 单轮：**给定 session + 本轮 message/argv**，返回文本与 **`continue` / `persist`** |
| **`executeCompleted`** | 编排已成功落库后：将本轮 assistant **`reply`**（及同源 **`metadata`**）回流外部系统 |
| **`decorateSessions`** | 列表展示：**会话维度** metadata，不参与单轮解析 |
| **`clearSession`** | 会话生命周期：删除/重置时插件侧兜底 |
| **`getScheduledTasks` / `runScheduledTask`** | 调度：`runScheduledTask` 仅 **`config`**；可内部调用 **`ingestExternalUserTurn`** 再次进入编排 |

**`runtime_plugin` 与 `command_plugin`**：前者可配合 **`sessionProvider.mode=multi`**；后者常为单会话或隔离命令上下文。**列表由 `decorateSessions` 塑形，回合由 `executeTurn` 处理**——二者无调用层级上的相交。

编排顺序与宿主表结构详见：[插件实例与编排](../../docs/插件/插件实例与编排.md)。

## 4. `PluginTurnContext` 与侧信道

宿主传入 **`executeTurn`** 的最小上下文包括：

- **`sessionId`、`message`、`config`**；可选 **`argv`**（宿主已解析命令名与参数）。
- **`emitAssistantDelta`、`emitPluginActivity`**：用于流式增量与可读阶段说明（如 `phase`、`data.summary`）。

插件 **不应假设** Hub 会自动转发上述侧信道——它们由宿主在 **`executeTurn`** 调用链上提供。

---

## 5. 清单驱动的能力门禁

宿主根据 **`plugin.json`**（尤其 **`capabilities` / permissions / `mcp.allowedServers`**）决定：

- 是否走 Chat、命令、`/command` 隔离上下文；
- 是否向 **`deps`** 注入 **`invokeHostLlm`、`invokeHostMcpTool`**；
- 是否在策略引擎中放行 MCP、跨插件调用、编排租约等。

SDK **只描述「若注入了该如何传参」**；不负责授权，授权在宿主。

## 6. SDK 源码模块角色（便于阅读代码）

| 模块 | 角色 |
|------|------|
| **`runtime-contract.ts`** | 全部公共类型与 **`PluginRuntimeExtension` / Module** |
| **`turn-result.ts` / `session-row.ts`** | 构造函数式规范化返回值与列表行 |
| **`host-mcp-guard.ts` / `host-llm-guard.ts`** | 在使用桥前统一检测注入与错误文案 |
| **`plugin-runtime-base.ts`** | 可选 **`BasePluginRuntime`**：封装 **`publish` / mcp / llm / ingest / workspace** |
