# `command_plugin` 开发检查清单（PR 评审用）

用途：面向 `kind = command_plugin` 的专项核验。  
说明：此清单是 `插件开发检查清单.md` 的补充，不替代通用检查。

---

## 1. 清单与能力声明

- [ ] `plugin.json.kind` 明确为 `command_plugin`。
- [ ] `capabilities.command` 与实际命令能力一致（启用即需可执行）。
- [ ] 若声明 `capabilities.chat = true`，已明确命令入口与普通消息入口边界。
- [ ] `triggerDescription`、`examples` 与真实命令语法一致。

---

## 2. 命令输入与参数处理

- [ ] `executeTurn` 对命令输入有稳定解析（优先使用 `ctx.argv`，缺失时回退解析 `ctx.message`）。
- [ ] 对空参数、非法参数、越界参数有统一错误提示。
- [ ] 命令帮助（如 `/help`）覆盖核心命令与参数示例。
- [ ] 命令执行失败不会导致整轮崩溃，能返回可读错误文本。

---

## 3. 返回协议与上下文写入

- [ ] 每次执行都返回合法 `PluginTurnHandleResult` 且包含 `text`。
- [ ] `continue` 的使用明确：命令执行完成后是否允许宿主继续编排。
- [ ] 若使用 `persist`，已确认写入消息角色与内容符合业务预期。
- [ ] 未返回宿主无法消费的私有字段作为主输出。

---

## 4. 命令副作用与幂等性

- [ ] 高风险命令具备防误触保护（确认、白名单或权限门槛）。
- [ ] 可重复执行命令具备幂等处理或冲突提示。
- [ ] 长耗时命令有阶段性 `emitAssistantDelta`（助手正文流）输出。
- [ ] 命令中断/超时后系统状态可恢复，不遗留脏状态。

---

## 5. 与宿主能力协作（可选）

- [ ] 使用 LLM 时对 `invokeHostLlm` 做 guard/判空。
- [ ] 使用 MCP 时对 `invokeHostMcpTool` 做 guard/判空。
- [ ] 若命令涉及外部进线回灌，`ingestExternalUserTurn` 调用已做错误分支处理。
- [ ] 宿主能力不可用时，命令仍能返回清晰降级提示。

---

## 6. 安全与审计

- [ ] 命令输入未直接拼接到危险执行路径（命令注入/路径注入防护）。
- [ ] 不输出敏感配置、凭据、token 到用户可见消息。
- [ ] 必要日志保留关键上下文（命令名、耗时、结果），避免泄露敏感信息。
- [ ] 权限声明最小化，与命令真实能力严格匹配。

---

## 7. 合并前结论

- [ ] 命令路径已完成最小联调（至少一条成功 + 一条失败分支）。
- [ ] 文档中的命令示例可直接复现。
- [ ] 我确认该 `command_plugin` 可在当前宿主契约下稳定运行。
