# 插件开发检查清单（PR 评审用）

用途：用于 `plugins/*` 插件在开发自检、代码评审、提测前的一次性核验。  
使用方式：逐项勾选，任一关键项未通过则不建议合并。

---

## 1. 入口与加载

- [ ] `plugin.json.entry` 指向真实存在的构建产物。
- [ ] 入口模块为 ESM，且使用 `export default class`。
- [ ] 入口默认导出可被宿主 `new` 成功（构造函数参数兼容 `PluginRuntimeExtensionDeps`）。
- [ ] 本地构建后可被 Node `import()` 正常加载。

---

## 2. 清单字段一致性（`plugin.json`）

- [ ] `id` 全局唯一且为 kebab-case。
- [ ] `kind` 为 `runtime_plugin` 或 `command_plugin`，与实现一致。
- [ ] `capabilities` 只声明已实现能力，无“虚假声明”。
- [ ] `permissions` 仅申请必要权限，无过度授权。
- [ ] `sessionProvider.mode` 与会话行为一致（`single`/`multi`）。
- [ ] `configSchema` 与 `defaultConfig` 相互匹配，默认值满足 schema 约束。

---

## 3. 契约与类型

- [ ] runtime 类实现遵循 `@wclaw/plugin-sdk` 契约（建议 `implements PluginRuntimeExtension`）。
- [ ] `executeTurn`（若实现）返回 `PluginTurnHandleResult`，且始终包含 `text`。
- [ ] `persist`（若返回）结构合法：`sessionId`、`role`、`content` 完整且可校验。
- [ ] `executeCompleted` / `decorateSessions` / `clearSession` / `getScheduledTasks` / `runScheduledTask`（若实现）签名正确。

---

## 4. 可选能力安全调用

- [ ] 对 `invokeHostLlm` 做了 guard/判空，不直接非空断言调用。
- [ ] 对 `invokeHostMcpTool` 做了 guard/判空，不直接非空断言调用。
- [ ] 对 `ingestExternalUserTurn`、`releaseHostMcpContext`（若使用）做了判空。
- [ ] 能力不可用时返回可理解的业务提示，不暴露底层错误堆栈给用户。

---

## 5. 架构边界与安全

- [ ] 未 `import` 宿主内部实现（如 `apps/host-api/**`）。
- [ ] 未直连 MCP Server（必须经宿主能力调用）。
- [ ] 未依赖宿主对 `pluginId` 的特判行为。
- [ ] 未引入与插件职责无关的高风险行为（无授权执行命令、无越权文件写入等）。

---

## 6. 调度任务（如适用）

- [ ] `getScheduledTasks` 返回的 `intervalMs`、`timeoutMs`、`maxRetry` 合理。
- [ ] `runScheduledTask` 单次执行可控，不会长期阻塞。
- [ ] 失败路径可重试或可观测，有基础日志/活动输出。
- [ ] 任务逻辑对配置缺失或异常输入有防御性处理。

---

## 7. 可观测与用户体验

- [ ] 关键流程使用 `emitPluginActivity` 输出必要进度信息（如开始/成功/失败）。
- [ ] 流式场景（如适用）使用 `emitAssistantDelta`，输出节奏稳定。
- [ ] 错误提示对业务方可读，不是仅返回 `Error: xxx`。
- [ ] 欢迎语、建议提示、命令帮助（如配置）与插件真实能力一致。

---

## 8. 测试与联调

- [ ] 至少完成一次端到端最小联调：进入会话并成功处理一轮输入。
- [ ] 覆盖关键失败路径：配置缺失、注入能力不可用、外部依赖失败。
- [ ] 如插件在 `lint:arch` 扫描范围内，已通过 `pnpm lint:arch`。
- [ ] 构建脚本和运行脚本在当前仓库可复现。

---

## 9. 合并前结论

- [ ] 我确认以上所有“关键项”均已通过。
- [ ] 我确认文档与实现一致，不存在“文档说支持但代码未实现”的情况。
- [ ] 我确认该插件可在不修改宿主内部代码的前提下独立演进。
