# 宿主-插件通信总线（HPC Bus）改造文档

## 0. 与《设计文档》的关系

- **产品需求表**：设计文档 **§1（通信中心需求整理）**、**§1.2 / §1.3**。
- **写死规则**：设计文档 **§0**（含 **§0.1.1** 对 `ctx.publish` 的允许范围）。

---

## 1. 改造目标（现行）

1. 落地 **`host-event-hub.service.ts`**：对外 **`publish`**、**`HOST_EVENT_TOPICS`**、按 topic **多路分发**、Sink 注册；**`dispatchNotificationToSubscribers` 仅经 Hub Sink**（§1.2.1）。
2. **`services/notification`**：薄封装 **`publishToNotificationStream`** → Hub **`HOST_EVENT_TOPICS.Notification`**。
3. **宿主生产者**：`scheduler-observer` 等只使用 **`publishToNotificationStream` / `publish`**，禁止直连 **`dispatchNotificationToSubscribers`**。
4. **`emitPluginActivity` / AI Chat SSE**：默认不改（§0.2）。
5. **插件注入 / 工厂**：不在本阶段（另行迭代）。

---

## 2. 文件与职责清单

| 动作 | 路径 |
|------|------|
| 新增 | `services/host-event-hub/host-event-hub.service.ts`（`publish`、`registerSink`） |
| 新增 | `services/host-event-hub/host-event-hub.topics.ts`（`HOST_EVENT_TOPICS`） |
| 新增 | `services/notification/notification-stream-publish.service.ts`（`publishToNotificationStream`） |
| 修改 | `notification-bus.service.ts`：对外投递改名为 **`dispatchNotificationToSubscribers`**，**唯一**由 Hub Sink 调用 |
| 修改 | `scheduler-observer.service.ts`：改为 **`publishToNotificationStream`** |
| 新增 | `providers/host-event-hub-provider/`（**`registerAllHostEventHubProviders`**、`notification-hub.provider`） |
| 修改 | `app.ts`：启动早期 **`registerAllHostEventHubProviders()`** |

**唯一入库（投递）规则**：凡进入常驻 SSE 订阅者的通知，必须先经 **`hostEventHub.publish`**（或 **`publishToNotificationStream`**），不得直连 **`dispatchNotificationToSubscribers`**。

### 本阶段不做

- 插件 **`publish` / 工厂导出**（后续单独迭代）。
