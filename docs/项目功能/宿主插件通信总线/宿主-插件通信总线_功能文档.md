# 宿主-插件通信总线（HPC Bus）功能文档

## 0. 硬性规则（与《设计文档》§0 一致）

- **同步调用**：宿主经 **`PluginRuntimePort.get`** 取得预加载实例（启动时 **`import` + `new` 默认类`**）→ 实例方法，**不经 Hub 回传返回值**（§0.1）。
- **`ctx.publish` 例外**：仅允许增加可选 **`publish?`**，不得再堆其它 HPC 字段（§0.1.1）。
- **`emitPluginActivity` + AI Chat SSE**：独立通道（§0.2）。
- **插件**：禁止 import 宿主内部、禁止直连 `notification-bus`；可用 **`ctx.publish`** 或胶水层（§0.4）。

---

## 1. 功能定位（需求摘要）

1. **通信中心**：`host-event-hub.service.ts`，统一 **`publish`**，按 **`HOST_EVENT_TOPICS`** 分发。
2. **Notification 对外 `publish`**：薄封装（如 **`publishToNotificationStream`**）→ Hub + **`Notification`** topic。
3. **插件**：通过 **`ctx.publish`**（宿主注入闭包）发事件，支持 **一次传多个 topic**，由 Hub 分发。
4. **与 AI Chat**：`emitPluginActivity` 仍管聊天流式；Hub `publish` 管常驻通知与其它 topic（如 **Chat**）。

全文口径与字段级说明见《宿主-插件通信总线_设计文档》**§1、§1.2、§1.3**。

---

## 2. 解决的问题

- 宿主与插件事件入口分散、难以按 topic 治理。
- 常驻通知与聊天流式混用同一概念。
- 缺少「多 topic 一次上报、中心分发」能力。

---

## 3. 核心能力

| 能力 | 说明 |
|------|------|
| **`hostEventHub.publish`** | 唯一推荐入站；支持 **`topics[]`**。 |
| **`HOST_EVENT_TOPICS`** | 常量层，如 **Notification**、**Chat**。 |
| **`publishToNotificationStream`** | Notification 专用语法糖，内部固定 Notification topic。 |
| **`ctx.publish`** | 插件侧入口，由 **plugin-chat / scheduler-runner** 注入。 |
| **Sink** | 按 topic 订阅；Notification Sink 仍落 **`publishNotification`**。 |

---

## 4. 功能范围

### 包含

- Hub、topic 常量、Notification 薄封装、调度迁线、**ctx.publish** 注入。

### 不包含（除非 ADR）

- 改写 **`emitPluginActivity` 默认行为**。
- 用 Hub 替代插件 **return**。

---

## 5. 背压与频率

见设计文档 §6；多 topic 扇出为同步循环时保持轻量。

---

## 6. 验收标准

- 设计文档 §1 表格与 §1.3 注入方案可落地、与代码一致。
- 旧插件不传 `publish` 仍运行；新插件可 **`publish({ topics: [Notification, Chat], ... })`**（以常量为准）。
