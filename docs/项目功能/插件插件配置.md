# 插件配置说明（以 weixin-bridge 为例）

本文基于当前文件 `plugins/weixin-bridge/plugin.json` 生成，目标是把插件清单字段的作用、默认值、约束和运行语义讲清楚，便于后续维护与扩展。

---

## 2. 顶层字段说明

| 字段 | 当前值 | 说明 |
|---|---|---|
| `id` | `weixin-bridge` | 插件唯一标识（kebab-case），用于路由、会话前缀、配置键。 |
| `displayName` | `微信桥` | 管理台展示名。 |
| `version` | `3.0.0` | 插件自身版本（与 `apiVersion` 不同）。 |
| `apiVersion` | `v3` | 插件规范版本，宿主按此校验。 |
| `kind` | `runtime_plugin` | 插件类型：运行时扩展。 |
| `entry` | `runtime.mjs` | 入口模块；需 `export default class`。 |
| `description` | 微信登录、多账号会话 | 插件用途说明。 |
| `triggerDescription` | 进入 Chat 后发送 /login | 触发说明。 |
| `examples` | `[{input:"/login"}]` | 示例输入。 |

---

## 3. 权限与能力

### 3.1 `permissions`

当前声明：
- `network.read`
- `network.write`
- `plugin.invoke`
- `mcp.call`

语义：插件涉及网络访问、插件编排调用及 MCP 代理调用时的权限申请。最终是否放行由宿主策略控制。

### 3.2 `capabilities`

| 字段 | 当前值 | 含义 |
|---|---|---|
| `chat` | `true` | 支持聊天入口（`handleChat`）。 |
| `llm` | `true` | 支持 LLM 参与编排。 |
| `command` | `true` | 支持命令执行能力。 |
| `commandContextWrite` | `result_only` | 命令结果对上下文写入策略。 |
| `isolatedContext` | `true` | 支持隔离上下文模式。 |
| `mcpAccess` | `policy_granted` | MCP 使用需策略放行。 |
| `crossPluginInvoke` | `policy_granted` | 跨插件调用需策略放行。 |
| `orchestration` | `runtime_lease` | 进入宿主编排租约治理。 |

---

## 4. 隔离会话控制（`isolation`）

| 字段 | 当前值 | 说明 |
|---|---|---|
| `exitCommands` | `["/close"]` | 显式退出隔离命令。 |
| `idleTimeoutSec` | `900` | 空闲超时（秒）。 |
| `maxTurns` | `60` | 最大轮次上限。 |

---

## 5. MCP 相关配置（`mcp`）

当前：

```json
{ "allowedTools": [] }
```

说明：
- 空数组不代表“禁用 MCP”，而是“当前未在清单里预绑定工具列表”；
- 实际可调用集合仍受宿主策略、插件配置和运行时注入共同控制。

---

## 6. 会话模型（`sessionProvider`）

当前：

```json
{ "mode": "multi" }
```

说明：
- 多会话模式；
- 微信桥会按 `default` + 账号会话组织 session；
- 默认会话用于登录引导，账号会话用于收发消息。

---

## 7. 交互引导（`guide`）

`guide` 负责前端会话欢迎文案与建议提示词，不参与权限控制。

### 7.1 通用引导
- `welcome`: 欢迎使用微信桥
- `suggestions`: `/help`

### 7.2 多会话引导（`multiSession`）
- `defaultSessionWelcome`: 默认会话的登录引导文案
- `sessionWelcome`: 账号会话文案
- `defaultSessionSuggestions`: 默认会话建议（`/help`、`/login`、`/accounts`）
- `sessionSuggestions`: 账号会话建议（`/send`、`/logout`）

---

## 8. 插件配置表单（`configSchema`）

宿主管理台会根据 `configSchema` 自动生成插件配置表单；保存值写入插件配置中心。

### 8.1 字段定义

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `autoReply` | boolean | - | 自动回复开关 |
| `pollIntervalMs` | number | 500~30000 | 轮询间隔 |
| `pollTimeoutMs` | number | 1000~20000 | 轮询超时 |
| `maxPollAccountsPerTick` | number | 1~20 | 单 tick 最大账号处理数 |
| `loginWaitTimeoutMs` | number | 10000~900000 | 登录等待超时 |
| `startupProbeEnabled` | boolean | - | 启动探活开关 |
| `startupProbeTo` | string | - | 探活目标 userId |
| `startupProbeText` | string | - | 探活文本 |

`required` 当前仅强制 `pollIntervalMs`。

### 8.2 默认值（`defaultConfig`）

用于初始化配置，避免插件首次运行缺省值不明确：
- `pollIntervalMs: 3000`
- `pollTimeoutMs: 5000`
- `maxPollAccountsPerTick: 5`
- `loginWaitTimeoutMs: 480000`
- 其余见上方完整 JSON。

---

## 9. 与运行时代码的映射关系

| 清单字段 | 运行时影响 |
|---|---|
| `entry` | 对应 `plugins/weixin-bridge/runtime.mjs`，宿主按默认类实例化。 |
| `capabilities.chat` | 宿主可调 `handleChat`。 |
| `sessionProvider.mode=multi` | 会话列表走多会话协议，`decorateSessions` 提供列表。 |
| `orchestration=runtime_lease` | 可接入宿主租约与编排治理。 |
| `guide.multiSession` | 前端按 default/account 会话展示不同欢迎语与建议词。 |
| `configSchema/defaultConfig` | 管理台渲染配置表单并保存。 |

---

## 10. 维护建议

1. 变更 `plugin.json` 时同步更新本文档（字段、默认值、约束）。
2. 若新增 `guide` 子字段（如插件层 `systemPrompt`），需同步更新：
   - `apps/host-api/src/core/plugin-manifest.types.ts`
   - `apps/host-api/src/core/validate-plugin-spec.ts`
   - 本文档对应章节
3. 涉及 MCP 能力时，建议明确记录：
   - 清单声明上限（`mcp.allowedTools`）
   - 运行时配置启用集合（插件配置）
   - 实际注入集合（策略过滤后）
