# 插件配置说明（以 weixin-bridge 为例）

本文以 `plugins/weixin-bridge/plugin.json` 当前内容为准，说明清单字段与运行语义。  
注意：会话欢迎文案/建议词已改为 `decorateSessions -> PluginSessionRow.ui` 提供，不再通过 `guide` 或 `sessionProvider`。

---

## 1. 完整配置（当前生效）

```json
{
  "id": "weixin-bridge",
  "displayName": "微信桥",
  "version": "3.0.0",
  "apiVersion": "v3",
  "kind": "runtime_plugin",
  "entry": "dist/runtime.mjs",
  "description": "微信登录、多账号会话（chat 内 /login）",
  "systemPrompt": "你是微信桥助手。优先帮助用户完成登录、会话切换、消息收发与常见故障排查，回答要简洁可执行。",
  "isolation": {
    "exitCommands": ["/close"],

  "mcp": { "allowedServers": ["playwright", "sequential-thinking"] },
```

---

## 2. 顶层字段说明

| 字段 | 当前值 | 说明 |
|---|---|---|
| `id` | `weixin-bridge` | 插件唯一标识（kebab-case）。 |
| `displayName` | `微信桥` | 管理台展示名。 |
| `version` | `3.0.0` | 插件版本。 |
| `apiVersion` | `v3` | 插件清单规范版本。 |
| `kind` | `runtime_plugin` | 运行时插件。 |
| `entry` | `dist/runtime.mjs` | 入口模块；需 `export default class`。 |
| `description` | 微信登录、多账号会话 | 插件用途。 |
| `systemPrompt` | 已配置 | 插件级系统提示词（命令插件上下文补全与相关编排路径会读取）。 |

---

## 3. 运行与隔离（`isolation`）

`isolation` 当前仅用于 `runtime_plugin` 场景。

| 字段 | 当前值 | 说明 |
|---|---|---|
| `exitCommands` | `["/close"]` | 显式退出隔离命令。 |
| `idleTimeoutSec` | `900` | 空闲超时（秒）。 |
| `maxTurns` | `60` | 最大轮次上限。 |

---

## 4. MCP 上限（`mcp.allowedServers`）

当前：

```json
{ "allowedServers": ["playwright", "sequential-thinking"] }
```

语义：声明插件可访问的 MCP server alias 上限。实际可调用工具仍受宿主策略过滤。

---

## 5. 配置表单（`configSchema` + `defaultConfig`）

管理台根据 `configSchema` 渲染表单，根据 `defaultConfig` 提供初始值。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `autoReply` | boolean | - | 自动回复开关 |
| `pollIntervalMs` | number | 500~30000 | 轮询间隔 |
| `pollTimeoutMs` | number | 1000~20000 | 轮询超时 |
| `maxPollAccountsPerTick` | number | 1~20 | 单 tick 最大账号处理数 |
| `loginWaitTimeoutMs` | number | 10000~900000 | 登录等待超时 |
| `startupProbeEnabled` | boolean | - | 启动探活开关 |
| `startupProbeTo` | string | - | 探活目标 userId |
| `startupProbeText` | string | - | 探活文本 |

---

## 6. 会话显示来源（最新约定）

- 清单不再承担会话欢迎文案配置（已移除 `guide` / `sessionProvider`）。
- 会话与引导 UI 由插件运行时 `decorateSessions()` 返回的 `PluginSessionRow` 决定：
  - 会话列表：`sessionId/title/updatedAt`
  - 引导区：`ui.welcome`、`ui.suggestions`
  - 其他显示：`ui.subtitle`、`ui.badges`、`ui.description` 等
- 前端规则：字段缺失即不显示，不做平台默认兜底。

---

## 7. 维护建议

1. 每次修改 `plugin.json` 后同步更新本文档。
2. 新增清单字段时，需同步更新：
   - `apps/host-api/src/core/plugin-object.types.ts`
   - `apps/host-api/src/schemas/plugin-spec-v3.schema.ts`
   - `apps/host-api/src/core/validate-plugin-spec.ts`
3. 若变更会话展示协议，优先更新 `packages/plugin-sdk/src/runtime-contract.ts` 与相关插件 `decorateSessions` 实现。
