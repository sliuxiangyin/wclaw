# 插件类型


| 插件类型             | 作用                                                             | 模式                                                                  | 说明                                                                                                                                                             |
| ---------------- | -------------------------------------------------------------- | ------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `runtime_plugin` | 运行时扩展（如微信桥），负责账号、消息、二维码、会话拉取等核心运行时能力。                          | 不适用（由运行时会话模型驱动）                                                     | 适合常驻运行与后台任务场景：可在 `setup(ctx)` 中注册轮询/定时任务（通过 `scheduler`），并通过 `emit` 持续向宿主上报事件。`runtime_plugin` 默认接入 LLM，并可在策略允许下编排其他插件能力。宿主可存在多个运行时扩展，但当前会优先选择第一个可用实例作为主运行时插件。 |
| `command_plugin` | 命令型插件，通过命令触发执行并返回结果。                                               | `commandMode` = `ephemeral_no_context` / `ephemeral_with_context` / `isolated_chat` | 适合问答、工具调用、流程触发等“按命令执行”的插件。执行语义由 `commandMode` 单字段决定。                                                                                                             |


## command_plugin 执行模式说明

> 约定：`ephemeral_no_context` / `ephemeral_with_context` / `isolated_chat` 是 `command_plugin.commandMode` 的执行模式。  
> 宿主 AI Chat 中：**显式斜杠命令**（`/command …` 或短 `/xxx`）才会调用插件 `executeTurn`。**无斜杠**时：`ephemeral_no_context` 只提示用法；`ephemeral_with_context` / `isolated_chat` 可按宿主编排**仅走 LLM**（带 `systemPrompt` 与历史），不先跑插件。细则见 **`docs/项目功能/插件/插件架构设计.md`**（`commandMode` 表）。

## command_plugin 三种模式（执行语义）


| 模式                       | 对应作用     | 介绍（简述）                                                                                 |
| ------------------------ | -------- | -------------------------------------------------------------------------------------- |
| `ephemeral_no_context`   | 无上下文命令执行 | 仅在命中插件命令时执行，不读取历史上下文；不接入 LLM。适合确定性、工具型、幂等命令。                                           |
| `ephemeral_with_context` | 带上下文命令执行 | **有**斜杠命令：先 `executeTurn`，再可按编排把命令输出交给 LLM；**无**斜杠：宿主可仅 LLM（与清单中的 MCP 声明一致），不先执行插件。                                     |
| `isolated_chat`          | 隔离会话命令执行 | 与 `ephemeral_with_context` 在「无斜杠 / 有斜杠」上对齐；从 runtime 会话经 `/command` 命中本模式时可进入 **隔离态**（见宿主编排）。隔离内 `/close` 退出。 |


## runtime_plugin（微信桥）补充

- `runtime_plugin` 不是上述三种 `command_plugin` 执行模式。
- `runtime_plugin` 通常接入 LLM，并承担常驻运行时职责（账号状态、会话拉取、后台任务）。
- 会话与引导展示由 `decorateSessions` 返回的 `PluginSessionRow` 驱动；插件不返回字段时前端不显示对应区域。
- 是否可编排其他插件由宿主策略与权限决定（插件清单不再通过 `capabilities.*` 多字段组合表达）。
- **硬性约束**：`runtime_plugin` 不可编排 `runtime_plugin`；`runtime_plugin` 仅可编排 `command_plugin`。
- 运行时编排治理由宿主统一负责（调度、租约、策略）。

## 术语对照（防混淆）


| 术语                                                                  | 适用对象                      | 含义                                   |
| ------------------------------------------------------------------- | ------------------------- | ------------------------------------ |
| `ephemeral_no_context` / `ephemeral_with_context` / `isolated_chat` | `command_plugin`          | 命令执行模式（何时执行、是否带上下文、是否走 LLM、是否隔离上下文）。 |
| `decorateSessions -> PluginSessionRow`                              | 运行时插件                     | 会话列表与会话引导 UI 的唯一数据来源。                |
| `kind=runtime_plugin`                                               | `runtime_plugin`          | 运行时插件，默认可进入 chat 运行时路径。              |
| `commandMode`                                                       | `command_plugin`          | 命令插件执行模式（单字段真值来源）。                   |


## executeTurn 返回协议（最新）

**适用所有 `kind`（`runtime_plugin` / `command_plugin`）**，由 `@wclaw/plugin-sdk` 的 `PluginTurnHandleResult` 与宿主统一实现。

- `executeTurn` 统一返回对象：`{ text, continue?, persist? }`，不再推荐返回裸字符串或 `reply` 字段。
- **默认值**（`toTurnResult` 与宿主一致）：
  - `continue` 缺省为 **`false`**：本轮 **`text` 即对用户可见的最终回复**（落库为一条助手消息等），宿主**不再**在同一轮编排里追加「再接宿主 LLM」等后续步骤（含 `command_plugin` 的 `ephemeral_with_context`：不再把输出先当正文再跑 LLM）。
  - `persist` 缺省为 **`[]`**。
- **`continue: true`（所有插件通用）**：`text` 视为**中间结果**，不作为本轮最终助手正文单独落库/展示终点；在宿主支持的路径上（例如 **`command_plugin` + `ephemeral_with_context` 且命中命令**）由宿主把 `text` **转写为发给宿主 LLM 的 user 消息**，**仅 LLM 输出**作为本轮对用户展示的助手回复。**`POST /api/plugins/:id/chat`（`callExecuteTurn`）**下为 `true` 时不写入本条「最终 assistant」行，避免与后续编排重复。
- **`persist`**：追加写入消息到指定会话（可跨会话），每项形如 `{ sessionId, role: "user" | "assistant", content }`，宿主校验 `sessionId` 归属当前插件。

### 示例

```js
return {
  text: `登录成功：${loggedInAccountId}。已创建会话「${accSessionId}」。`,
  continue: false,
  persist: [
    {
      sessionId: accSessionId,
      role: "assistant",
      content: `账号 ${loggedInAccountId} 已连接。发送 /send <联系人> <内容> 可发消息。`
    }
  ]
};
```
