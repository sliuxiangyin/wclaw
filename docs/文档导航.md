## 2. 文档分工

- `packages/plugin-sdk/README.md`（仓库内）：**`@wclaw/plugin-sdk`** 包说明与插件侧引用方式（类型契约；**weixin-bridge 仅外层 runtime**，**不**改 `openclaw-weixin` 子项目）。
- `目录架构.md`：先建哪些目录、每层放什么。
- `设计模式与分层规范.md`：代码如何拆层、避免写成一锅粥。
- `scripts/check-architecture.mjs`：哪些架构规则会被脚本拦截。
- `进度/任务TODO.md`：今天要做什么、优先级是什么。
- `进度/外部进线-ingest检查清单.md`：外部进线 P0/P1 与微信 `poll-inbox` 联动验收项。
- `weixin_bridge_api_contract_微信桥接口契约.md`：微信桥 HTTP/Chat/调度/进线/回流字段约定（无宿主专用路由）。
- `进度/功能清单_status.md`：已实现与未完成能力的统一对照表。
- `进度/任务TODO.md` 与 `进度/功能清单_status.md`：两者需联动维护，避免状态漂移。
- `项目功能/宿主/宿主功能.md`：宿主能力统一口径（插件/调度/LLM/Chat）。
- `项目功能/插件/插件架构设计.md`：插件架构总览（类型职责、生命周期、调用链、安全边界、演进策略）。
- `Scheduler调度器_需求与设计方案.md`：Scheduler 专项需求、模型、流程与分阶段实施基线。
- `项目功能/插件插件配置.md`：插件配置字段说明、能力声明与宿主运行映射。
- `notification_sse_长连接通知通道设计.md`：Notification SSE（**Host Event Hub 的 Sink 之一**）设计与连接治理。
- `项目功能/宿主插件通信总线/宿主-插件通信总线_功能文档.md`：Ingress + Hub + 多 Sink 的功能边界。
- `项目功能/宿主插件通信总线/宿主-插件通信总线_设计文档.md`：**§0 写死规则**（`loadPluginRuntimeExtension` / `emitPluginActivity`+AI Chat SSE / Hub 与胶水层）、事件模型、扇出、背压。
- `项目功能/宿主插件通信总线/宿主-插件通信总线_改造文档.md`：Hub 引入、Ingress 接线、按文件拆解的阶段计划。
- `项目功能/MCP服务/README.md`：**宿主 MCP Gateway**（`mcpServers` 配置、stdio/http、`/api/mcp/*` 与控制台列表及 JSON 编辑页）索引；正文见 `MCP_Gateway_需求与设计建议.md`。
- `项目蓝图.md`：总体原则与技术选型。
- `项目功能/插件插件配置.md`：插件字段与配置说明（以现有插件为样例）。

## 3. 维护规则（简化）

1. 新需求先改文档，再改代码。
2. API 变更必须同步更新对应契约文档。
3. TODO 每周至少更新一次状态。
