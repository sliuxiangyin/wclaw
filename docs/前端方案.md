# v3 前端方案（宿主控制台）

本文用于同步当前版本的前端实现基线与下一阶段规划，统一对齐宿主四大能力：

- 插件（Plugin）
- 调度与编排（Scheduler / Orchestration）
- LLM 配置（LLM Config）
- Chat（会话与消息流）

---

## 1. 当前路由基线（按代码现状）

```txt
/                     -> 重定向到 /plugins
/plugins              -> 插件列表页
/settings/llm         -> LLM 配置页
/chat/:pluginId       -> 插件 Chat 工作区
```

说明：

- 当前没有 `/plugins/:pluginId/chat`，统一使用 `/chat/:pluginId`。
- MCP 设置页、编排可视化页尚未落地。

---

## 2. 模块能力与页面映射

### 2.1 插件（Plugin）

对应页面：`/plugins`

当前已支持：

- 插件列表展示（Grid）
- 从卡片进入 Chat
- 打开配置抽屉并保存配置
- 配置校验接口调用（后端当前为占位校验）

建议补强：

- 搜索与筛选（kind / capability / status）
- 卡片状态徽标（chat / llm / mcp / orchestration）
- 最近异常提示与“查看详情”入口

### 2.2 调度与编排（Scheduler / Orchestration）

对应页面：当前无独立页面（先在插件页与 Chat 页提供轻量可视化）。

当前状态：

- 前端暂未提供编排租约、调度任务、重试队列可视化。

建议补强：

- 在插件卡片增加“运行态摘要”：任务数、最近重试、熔断状态。
- 在 Chat 页增加“会话模式标签”：`normal` / `isolated`。
- 后续新增 `/settings/orchestration`（或 `/ops/orchestration`）集中管理。

### 2.3 LLM 配置（LLM Config）

对应页面：`/settings/llm`

当前已支持：

- 获取和保存宿主 LLM 配置（`GET/PUT /api/llm/config`）
- 基础表单编辑流程

建议补强：

- 表单分组：连接配置 / 生成参数 / 超时与流式策略
- 配置保存前本地校验（必填、数值范围）
- 插件级覆盖策略展示（先只读提示，后续再开放编辑）

### 2.4 Chat（会话与消息流）

对应页面：`/chat/:pluginId`

当前已支持：

- `assistant-ui` 消息交互
- 会话列表拉取与切换
- `chat` 与 `command` 两类消息调用
- 基础错误反馈与状态展示

建议补强：

- SSE 事件态可视化（连接中/重连中/失败）
- 工具调用消息的统一渲染样式
- 系统消息和插件消息的来源标记

## 4. UI 组件与分层约束

组件职责：

- `assistant-ui`：消息区与输入区
- 页面壳（`pages/*`）：只做编排与路由
- 业务状态（`features/*/hooks`）：请求、状态机、交互流程
- API 调用（`lib/api/*`）：统一封装，页面层禁止直接 `fetch`

硬约束：

- 前端只调用宿主 API，不调用插件内部端点。
- 不允许按插件 ID 写特判 UI 分支（如 `pluginId === "weixin-bridge"`）。
- 视图行为依赖 `PluginSpecV3` 与会话数据，不绑定插件实现细节。
