# 数据库

本文档描述 `apps/host-api` 当前使用的 SQLite 数据库结构（`var/data/host.db`）及各表用途。

## 概览

当前共 7 张业务表：

- `plugin_configs`
- `llm_config`
- `plugin_chat_messages`
- `chat_sessions`
- `chat_events`
- `plugin_chat_activity`
- `mcp_servers`

---

## 1) `plugin_configs`

**用途**

- 存储插件配置（按 `plugin_id` 一条记录）。

**核心字段**

- `plugin_id` (TEXT, PK): 插件唯一标识
- `config_json` (TEXT, NOT NULL): 插件配置 JSON
- `updated_at` (TEXT, NOT NULL): 更新时间（ISO 字符串）

---

## 2) `llm_config`

**用途**

- 存储 LLM 配置（按 `scope` 维度一条记录）。

**核心字段**

- `scope` (TEXT, PK): 配置作用域
- `config_json` (TEXT, NOT NULL): LLM 配置 JSON
- `updated_at` (TEXT, NOT NULL): 更新时间（ISO 字符串）

---

## 3) `plugin_chat_messages`

**用途**

- 插件聊天消息主表，记录 user/assistant 消息内容与来源属性。

**核心字段**

- `id` (INTEGER, PK AUTOINCREMENT): 自增主键
- `plugin_id` (TEXT, NOT NULL): 插件 ID
- `session_id` (TEXT, NOT NULL): 会话 ID
- `trace_id` (TEXT): 链路追踪 ID
- `role` (TEXT, NOT NULL): `user` / `assistant`
- `content` (TEXT, NOT NULL): 消息正文
- `source_type` (TEXT, DEFAULT `'runtime'`): 来源类型（如 runtime/plugin）
- `source_plugin_id` (TEXT): 来源插件 ID
- `llm_eligible` (INTEGER, DEFAULT 1): 是否可参与 LLM 上下文
- `context_summary` (TEXT): 上下文摘要
- `created_at` (TEXT, NOT NULL): 创建时间（ISO 字符串）

**索引**

- `idx_plugin_chat_messages_session_trace(plugin_id, session_id, trace_id)`

**备注**

- 该表通过多次 `ALTER TABLE` 进行过增量字段补齐（兼容历史库）。

---

## 4) `chat_sessions`

**用途**

- 存储会话级状态（如 normal/isolated 模式）以及 MCP 工具禁用策略。

**核心字段**

- `plugin_id` (TEXT, NOT NULL): 插件 ID
- `session_id` (TEXT, NOT NULL): 会话 ID
- `mode` (TEXT, NOT NULL): 会话模式（如 `normal` / `isolated`）
- `isolated_plugin_id` (TEXT): 隔离模式对应插件 ID
- `mcp_tool_forbidden` (TEXT): MCP 工具禁用策略 JSON（支持禁用整个 server 或 server 下指定 tool）
- `updated_at` (TEXT, NOT NULL): 更新时间（ISO 字符串）

**主键**

- 复合主键：`PRIMARY KEY (plugin_id, session_id)`

---

## 5) `chat_events`

**用途**

- 存储聊天事件流水，用于事件查询与审计。

**核心字段**

- `id` (INTEGER, PK AUTOINCREMENT): 自增主键
- `trace_id` (TEXT): 链路追踪 ID
- `plugin_id` (TEXT, NOT NULL): 插件 ID
- `session_id` (TEXT, NOT NULL): 会话 ID
- `type` (TEXT, NOT NULL): 事件类型
- `source` (TEXT, NOT NULL): 事件来源
- `payload_json` (TEXT, NOT NULL): 事件载荷 JSON
- `created_at` (TEXT, NOT NULL): 创建时间（ISO 字符串）

---

## 6) `plugin_chat_activity`

**用途**

- 存储插件活动（phase）时间线，支持会话时间线拼接与流式活动回放。

**核心字段**

- `id` (INTEGER, PK AUTOINCREMENT): 自增主键
- `plugin_id` (TEXT, NOT NULL): 插件 ID
- `session_id` (TEXT, NOT NULL): 会话 ID
- `trace_id` (TEXT, NOT NULL): 链路追踪 ID
- `seq` (INTEGER, NOT NULL): 同一 trace 下活动顺序
- `phase` (TEXT, NOT NULL): 活动阶段
- `payload_json` (TEXT, NOT NULL): 活动载荷 JSON
- `created_at` (TEXT, NOT NULL): 创建时间（ISO 字符串）

**索引**

- `idx_plugin_chat_activity_session(plugin_id, session_id)`
- `idx_plugin_chat_activity_trace(trace_id)`

---

## 7) `mcp_servers`

**用途**

- 存储 MCP Server 配置与探测状态快照。

**核心字段**

- `id` (TEXT, PK): MCP Server ID
- `config_json` (TEXT, NOT NULL): 服务配置 JSON
- `status_json` (TEXT, NOT NULL): 探测状态与工具快照 JSON
- `updated_at` (TEXT, NOT NULL): 更新时间（ISO 字符串）

**备注**

- 首次启动时若不存在 `playwright`，会自动插入默认记录。

---

## 初始化与位置

- 代码位置：`apps/host-api/src/core/db.ts`
- 数据库文件：`apps/host-api/var/data/host.db`（在仓库根执行 `pnpm dev:api` 的常见场景）
